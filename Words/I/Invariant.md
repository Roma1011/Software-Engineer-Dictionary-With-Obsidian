
ნიშნავს **პირობას, წესს ან თვისებას, რომელიც ყოველთვის უნდა იყოს დაცული და არ უნდა შეიცვალოს სისტემის მუშაობის დროს**.


მაგალითად:

ბანკის ანგარიშში შეიძლება არსებობდეს ასეთი invariant:

- ანგარიშის ბალანსი **არ შეიძლება იყოს უარყოფითი**

```
balance >= 0
```


DDD-ში **invariants** არის **ბიზნეს წესები**, რომლებიც უნდა იყოს დაცული **aggregate-ის შიგნით**.

მაგალითად:

Order aggregate-ში invariant შეიძლება იყოს:

- Order არ შეიძლება იყოს `Paid`, თუ ჯერ `Created` არ იყო
- OrderItem quantity ყოველთვის უნდა იყოს **> 0**

ანუ aggregate პასუხისმგებელია იმაზე, რომ ეს წესები **არასოდეს დაირღვეს**.
