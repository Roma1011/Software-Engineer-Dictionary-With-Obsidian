**Backfill** ნიშნავს **არსებული (ძველი) მონაცემების შევსებას ან გადამუშავებას ახალი წესის, სქემის ან ლოგიკის მიხედვით**.

	**Backfill = წარსული მონაცემების “მოყვანა დღევანდელ მდგომარეობაში”**

## განმარტება

**Backfill** გამოიყენება მაშინ, როცა:

- დაემატა ახალი ველი / სქემა
    
- შეიცვალა ბიზნეს ლოგიკა
    
- გაჩნდა ახალი ანალიტიკური მოთხოვნა
    
- საჭიროა ისტორიული მონაცემების გათანაბრება ახალ სტანდარტთან


## პრაქტიკული მაგალითები

### 1️ Database Schema

დავამატეთ ახალი ველი:

	`ALTER TABLE users ADD COLUMN country VARCHAR(50);`

 ძველ ჩანაწერებს `country` არ აქვთ → საჭიროა **backfill**:

	`UPDATE users SET country = 'GE' WHERE country IS NULL;`

### 2 Event / Schema Evolution

- დაემატა ახალი field event-ში (`createdAt`)
    
- ახალი consumer მას ელის
    

 ძველი event-ები უნდა **backfill**-დეს (replayed ან recalculated).

### 3 Analytics / Metrics

- დაემატა ახალი KPI
    
- ისტორიულ რეპორტებში ცარიელია
    

წარსული მონაცემებიდან ხდება **backfill**, რომ გრაფიკი სრულად იყოს შევსებული.


### 4 ETL / Data Pipeline

- შეიცვალა Transform ლოგიკა
    
- ძველი მონაცემები არ შეესაბამება ახალ ფორმატს
    

ხდება **backfill job**, რომელიც გადის ისტორიულ მონაცემებზე.


### Backfill vs Migration

|ტერმინი|რას ნიშნავს|
|---|---|
|**Migration**|სტრუქტურის შეცვლა (schema, table, column)|
|**Backfill**|ძველი მონაცემების შევსება ახალ სტრუქტურაში|
