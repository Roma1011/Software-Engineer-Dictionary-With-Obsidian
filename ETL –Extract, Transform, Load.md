ETL არის მონაცემთა მართვის პროცესის ნიმუში, რომელიც გამოიყენება **მონაცემების ანალიზისა და საცავებში (Data Warehouse) შესატანად**.


##  კომპონენტები

1. **Extract (ამოღება)**
    
    - მონაცემების მოპოვება სხვადასხვა წყაროებიდან: relational DB, CSV, API, log ფაილები.
        
    - მიზანია ყველა საჭირო მონაცემის შეკრება ერთ სივრცეში.


2. **Transform (გადამუშავება/გადაკეთება)**
    
    - მონაცემების კონვერტაცია ან გაწმენდა:
        
        - სტრუქტურის შეცვლა (normalization/denormalization)
            
        - ტიპების შეცვლა (string → date, float → decimal)
            
        - ბიზნეს-ლოგიკის გამოყენება (მაგ., ფასების კონვერტაცია სხვადასხვა ვალუტაში)


3. **Load (ჩატვირთვა)**
    
    - მზად მონაცემთა ჩატვირთვა საბოლოო დანიშნულების ადგილას:
        
        - Data Warehouse (DW)
            
        - Data Lake
            
        - ანაბეჭდული ან ალგორითმული სისტემები


## მაგალითი

თქვენი კომპანია იღებს მონაცემებს:

- e-commerce DB (orders, users)
    
- CSV ფაილები marketing campaign-ების შესახებ
    
- External API მონაცემები stock და ფასების შესახებ
    

**ETL პროცესი:**

1. **Extract** – ყველა წყაროს მონაცემები მოგროვდება.
    
2. **Transform** – მონაცემები გაწმენდილია, ვალიდაცია, ერთიანი სტრუქტურა, დროის ზონების კონვერტაცია.
    
3. **Load** – ჩატვირთვა Data Warehouse-ში, რომ BI ანალიტიკისთვის გამოიყენონ.

## გამოყენების მიზანი

- მონაცემთა გაერთიანება სხვადასხვა სისტემიდან
    
- მაღალი ხარისხის, სუფთა მონაცემების უზრუნველყოფა ანალიტიკისთვის
    
- Historical/aggregated data-ის შენახვა BI dashboards, reporting, Machine Learning-სთვის
